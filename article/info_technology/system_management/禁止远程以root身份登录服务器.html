# 禁止远程以 root 身份登录服务器

2020年04月09日 - YuweiYin

出于系统的安全性考虑，允许远程用户以 root 身份登录服务器是比较危险的。因此往往会禁止远程 root 登录，但同时应保留“跳板”用户得以访问服务器，并能够获得足够的权限。

# 1. 新增用户

查看当前所有用户：# cat /etc/group

增添新用户：# adduser admin

修改用户密码（不要太过简单）：# passwd admin

# 2. 赋予新用户 sudo 权限

当用户执行 $ sudo command 时，系统会寻找 /etc/sudoers 文件，判断该用户是否具备执行 sudo 命令的权限。

修改 /etc/sudoers 文件，添加一行，为新用户 admin 赋予全部权限：

# vim /etc/sudoers

增添 admin  ALL=(ALL)  ALL

# 3. 禁止远程 root 身份登录

修改 /etc/ssh/sshd_config 文件：

# vim /etc/ssh/sshd_config

找到 PermitRootLogin yes
改为 PermitRootLogin no

重启 ssh 服务（不同服务器可能重启方式不同）：# service sshd restart

# 4. 重启 sshd 服务可能报错

如果报错：sshd: unrecognized service

查看本机 Linux 发行版：# uname -a

如果是 Ubuntu，安装 ssh 相关服务：# sudo apt-get install openssh-client openssh-server

重启服务：# service sshd restart

如果 service 命令仍然无法运行，可考虑直接使用 /etc/init.d 下的 ssh 命令：# /etc/init.d/ssh restart

# 至此，设置便成功。

如果还试图远程登录 root 用户，无论是否正确输入 root 密码，都会报错：Permission denied, please try again.

只需用 admin 账号登录，登录之后使用 $ sudo 以 root 权限执行命令即可。

或者以 $ su 命令登录 root 用户（需要输入 root 密码）。
