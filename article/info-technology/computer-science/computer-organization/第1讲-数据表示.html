1.1 冯·诺依曼体系结构

  • 美籍匈牙利数学家冯·诺依曼总结了计算机的设计思想，提出了冯·诺依曼体系结构，包括以下三个方面：
    1. 以二进制数字代码的形式 表示信息 (数据) 和对信息的处理流程 (指令)。
    2. 采用存储程序的工作方式。(程序 即是预先存储好的 指令集)
    3. 计算机硬件系统由 “运算器、控制器、存储器、输入设备、输出设备” 等五大部件组成。

  • John von Neumann
    • 冯·诺依曼 1903 年出生，1921 年至 1925 年他先后在柏林和苏黎世学习化学，1926 年获得苏黎世化学工程文凭和布达佩斯数学博士证书。1930 年他以客座讲师身份到美国普林斯顿大学讲学，次年应聘为普林斯顿大学教授。
    • 冯·诺依曼对 ENIAC 机 (Electronic Numerical Integrator and Computer)不足之处进行认真分析，并讨论全新的存储程序的通用计算机方案。当军方要求比 ENIAC 性能更好的计算机时，他便提出 EDVAC（埃德瓦克）方案 (Electronic Discrete Variable Automatic Computer)。
    • 1946 年 6 月冯·诺依曼与戈德斯坦（H.Goldstine）等发表了《电子计算机装置逻辑结构初探》的论文，成为 EDVAC 的设计基础。1952 年完成了 EDVAC 机的建造工作，并投入运行，用于核武器的理论计算。
  • EDVAC 是电子离散变量计算机的缩写。它的主要改进有两点：
    • 改进一：为了充分发挥电子元件的高速性能而采用了二进制；
    • 改进二：把指令和数据都存储起来，让机器能自动地执行程序。
    • 由于它利用水银延时线作主存，用磁鼓作辅存，其运算速度比 ENIAC 提高了 240 倍。

# 用二进制数字代码表示信息

  • 二进制是计算技术中广泛采用的一种数制，二进制数据是用 0 和 1 两个数码来表示的数。
  • 由德国数理哲学大师 莱布尼兹 (Gottfried Wilhelm Leibniz, 1646.07.01 - 1716.11.14) 首创。
  • 19 世纪爱尔兰逻辑学家 乔治·布尔 (George Boole, 1815.11.02 - 1864.12.08) 将 对逻辑命题的思考过程 转化为 对符号 0 和 1 的代数演算 (布尔代数)，二进制是逢 2 进位的进位制，而 0 和 1 是其中的基本算符。
  • 由于二进制只使用 0、1 两个数字符号，代表了两种状态，易于用电子方式实现 (门电路的开/关)。

# 存储程序工作方式

  • 存储程序是冯．诺依曼思想的核心内容，包含三个要点：
    • 根据求解问题事先编制程序；
    • 事先将程序存入计算机中；
    • 计算机自动、连续地执行程序。
  • PC (Program Counter) 程序计数器
    • 用于存放当前正在执行的指令所在的存储单元的地址。
    • 每执行完一条指令后，PC 值自动加一，从而指向下一条指令，完成顺序执行。

1.3 数据信息的表示方法

# 数据编码与表示

  • 计算机中要存储的数据
    • 程序、整数、浮点数、字符(串)、逻辑值
    • 需要编码进行表示
  • 存储方式
    • 用数字电路的状态表示
    • 由上一层的抽象计算机来识别不同的内容
  • 编码原则
    • 少量简单的基本符号
    • 一定的规则
    • 表示大量复杂的信息

# 二进制码(基二码)数据表示

  • 使用两个基本符号: 0, 1
    • 符号个数最少，物理上容易实现
    • 与二值逻辑的 真/假 两个值相对应
    • 用二进制码表示数值数据运算规则简单
  • 二进制位可以表示任何对象:
    • 字符 (各种编码的各种字符)
    • 数值 (整数、小数/浮点数、正负数等)
    • 逻辑值 (0 或者 1)
    • 颜色 (RGB)
    • 位置 / 地址 / 指令
  • 但 n 位至多只能表示 2^n 个不同的对象

# 用二进制数字代码表示数值型数据

  • 简单例子：
    • +5: 00101
    • -6: 10110
  • 带符号数、不带符号数
  • 定点数、浮点数
  • 原码、反码、补码表示法
  • 运算法则
  • 规格化
  • 溢出判断

# 用二进制数字代码表示逻辑型数据

  • 逻辑型数据只有两个值：真/假。
    • 正好可以用二进制码的两个符号分别表示。
    • 例如：1 表示 真，0 表示 假。
    • 不必使用另外的编码规则。
  • 对逻辑型数据可以执行逻辑的 and 与、or 或、not 非 等基本逻辑运算。其规则如下：

# 用二进制数字代码表示字符型数据

  • 字符作为人机联系的媒介，是重要的数据类型之一。
  • 英文字符集由 128 个符号组成，通常用 8 位二进制编码，即用一个字节 Byte 来表示每一个符号。
  • 当前通用且常用的标准字符集：
    • ASCII 全称 American Standard Code for Information Interchange)
      • ASCII 码使用指定的 7 位或 8 位二进制数组合来表示 128 或 256 种可能的字符。
      • 标准 ASCII 码也叫基础 ASCII 码，使用 7 位二进制数 (剩下的 1 位二进制为 0) 来表示所有的大写和小写字母，数字 0 到 9、标点符号，以及在英语中使用的特殊控制字符。
    • Unicode 使用 16 位表示一个字符，可以表示 65536 个字符
      • 将整个编码空间划分为块，每块为16的整数倍，按块进行分配
      • 保留6400个码点供本地化使用，但仍无法覆盖所有字符
    • ISO/IEC 10646 全称 Information technology -- Universal Coded Character Set (UCS)

例如：(101.01)_{2} = 1 * 2^2 + 0 * 2^1 + 1 * 2^0 + 0 * 2^{-1} + 1 * 2^{-2} = (5.25)_{10}

八进制数的表示

  • 每位可选用的数码有 8 个: {0, 1, 2, 3, 4, 5, 6, 7}
  • 逢 8 进位，即基数为 8

例如：(703.64)_{2} = 7 * 8^2 + 0 * 8^1 + 3 * 8^0 + 6 * 8^{-1} + 4 * 8^{-2} = (451.8125)_{10}

十六进制数的表示

  • 每位可选用的数码有 16 个: {0, 1, ..., 9, A, B, C, D, E, F}
  • 逢 16 进位，即基数为 16

例如：(B89.64)_{2} = 11 * 16^2 + 8 * 16^1 + 9 * 16^0 + 6 * 16^{-1} + 4 * 16^{-2} = (2954.390625)_{10}

# 进制转换

十进制转二进制

从二进制数求其十进制的值，逐位码权累加求和

例：(11.625)_{10}

二进制到八进制或十六进制转换

  • 二进制转八进制：从小数点向左及向右，每三位一分组

    • 例如：(10 011 100 . 01)_{2} = (234 . 2)_{8}

  • 二进制转十六进制：从小数点向左及向右，每四位一分组

    • 例如：(1001 1100 . 01)_{2} = (9C . 4)_{16}

  • 注意：

    • 整数部分不足位数，默认在最左补零，即便不补也对对转换结果无影响；
    • 但小数部分不足位数，要在最右补零，否则出错。

1.5 数值数据编码表示

# 数值数据在计算机内的格式

  • 整数 (Integer Number)
    • $N = N_{s} N_{n} N_{n-1} ... N_{1} N_{0}$
    • s 意为 sign 符号，$N_s$ 是符号位
    • 下标 n .. 0 越大越靠左
  • 定点小数 (Fixed Point Number) (整数部分为 0)
    • $N = N_{s} N_{-1} N_{-2} ... N_{-n}$
  • 浮点数 (Floating Point Number) (IEEE 754 标准)
    • $N = M_{s} E_{s} E_{m-1} ... E_{0} M_{-1} M_{-2} ... M_{-n}$
    • M 意为 Mantissa 尾数，以原码表示
    • E 意为 Exponent 阶码，用作移位
    • 设浮点数的基数为 r (常取 2, 8, 16)，则 N = M * r^E
浮点数类型 符号位 阶码位 尾数数码位 总位数
短浮点数 1 8 23 32
长浮点数 1 11 52 64
临时浮点数 1 15 64 80
  • 带符号数的表示：
    • 真值：日常书写中，用正、负符号 (+/-) 加绝对值 表示数值，这种形式表示的数值称为真值。
    • 机器数：计算机内部 连同符号都一起数字化(二值化) 的数，称为机器数。
    • 例如：
      • 真值 +0.1001 机器数 0..1001 (小数点在倒数第四个数码前)
      • 真值 -0.1001 机器数 1..1001 (小数点在倒数第四个数码前)
      • 真值 +1001 机器数 0..1001 (小数点在末尾)
      • 真值 +1001 机器数 1..1001 (小数点在末尾)
